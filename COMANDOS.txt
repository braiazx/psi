========================================
COMANDOS ÚTEIS - ORDENATE
========================================

ABRIR PASTAS (WINDOWS):
-----------------------
# Abrir o explorador de arquivos na pasta atual
explorer .

# Abrir VS Code na pasta atual (se tiver instalado)
code .

# Abrir pasta específica no explorador
explorer C:\Users\478113\Desktop\projeto001

# Voltar para pasta anterior
cd ..

# Ir para pasta do projeto (raiz)
cd C:\Users\478113\Desktop\projeto001


INSTALAÇÃO:
-----------
# Instalar dependências na raiz
npm install

# Instalar dependências do backend
cd backend
npm install

# Instalar dependências do frontend
cd frontend
npm install


DESENVOLVIMENTO:
----------------
# Iniciar frontend e backend simultaneamente (na raiz)
npm run dev

# Iniciar apenas o frontend
cd frontend
npm run dev

# Iniciar apenas o backend
cd backend
npm run dev


BUILD (PRODUÇÃO):
-----------------
# Build de frontend e backend (na raiz)
npm run build

# Build apenas do frontend
cd frontend
npm run build

# Build apenas do backend
cd backend
npm run build


EXECUTAR EM PRODUÇÃO:
---------------------
# Executar backend em produção
cd backend
npm start

# Executar frontend em produção
cd frontend
npm start


BANCO DE DADOS (PRISMA):
------------------------
# Gerar cliente Prisma
cd backend
npm run prisma:generate

# Criar/executar migrações
cd backend
npm run prisma:migrate

# Abrir Prisma Studio (interface visual do banco)
cd backend
npm run prisma:studio


LINT (VERIFICAR CÓDIGO):
------------------------
# Verificar código de frontend e backend (na raiz)
npm run lint

# Verificar apenas backend
cd backend
npm run lint

# Verificar apenas frontend
cd frontend
npm run lint


OBSERVAÇÕES:
------------
- O backend roda por padrão na porta 3001
- O frontend roda por padrão na porta 3000
- Use 'npm run dev' na raiz para iniciar ambos simultaneamente
- Sempre execute 'npm install' após clonar ou atualizar o projeto
